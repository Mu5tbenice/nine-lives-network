- [ ] 16.0 Cinematic battle playback (est: XL ‚Äî feature work) **Status: deferred (Phase 3 backlog).** End-of-session brainstorm captured 2026-04-19 during the PR #151 cycle. Concept and rationale below.

  **Concept.** The combat engine already resolves rounds deterministically on the server. Capture the full per-round event log to a new `battle_logs` table; play it back to clients as a curated cinematic *after* the round resolves rather than rendering it live. Dramatic camera cuts, slow-motion on critical moments (final-blow KO, comeback flips, multi-kills), Nerm narration overlays, optional generative voice/text per round outcome.

  **Aesthetic reference.** AI-agent UIs like Devin and Replit Agent ‚Äî the user watches the system "perform" work in a deliberately readable, visually engaging frame. Auto-battler combat that's tuned for *spectacle replay* rather than real-time legibility opens a different design space than what live arena rendering allows.

  **Architecture.**
  - **Capture (16.1, low cost):** server-side append to `battle_logs` table on every relevant tick event (KO, crit, effect_applied, round_end). Schema mirrors the existing socket payloads. Write-only at first.
  - **Playback (16.0 main, high cost):** client-side cinematic engine consumes the log. Rule-based pattern matcher decides shot framing ‚Äî e.g., "if KO within 2s of crit ‚Üí slow-mo wide-angle hold on victim", "if guild flips control ‚Üí sweep camera across newly-aligned zone tiles", "if Nine kills 3+ in a round ‚Üí trigger 'dominance' montage". Roughly zero runtime AI/inference cost.
  - **Optional generative narration:** async LLM call after round_end produces a 1-2 sentence Nerm voice-over per round, cached per `battle_logs.id`. Marginal API cost; gates cleanly behind a flag.

  **Art cost (the real number).** Significant. Requires layered animated sprites for NFT-driven trait combinations (run cycle, attack anim, KO anim, hit reaction) ‚Äî multiplied across 9 houses √ó house-driven base loadout. Most expensive single line item in the proposal. Should be estimated separately before committing to 16.0 itself.

  **Unlock value.**
  - **Shareable highlight clips for Twitter** ‚Äî each round becomes a 15‚Äì30 sec exportable clip, organic distribution flywheel.
  - **Aesthetic differentiation** vs every other auto-battler in the genre, most of which struggle with "live combat is illegible because too much is happening at once".
  - **Frees up live-mode design constraints** ‚Äî once cinematic playback exists, live arena no longer has to be "the canonical viewing experience" and can be designed for tactical input clarity instead.
  - **Reusable for marketing/onboarding** ‚Äî generated highlight reels become onboarding content, social proof, and first-impression material for new players.

  Phase: 3. Est: XL. Status: deferred ‚Äî capture this brainstorm so it's not lost; revisit when Phase 2/3 stakeholder priorities are set.
